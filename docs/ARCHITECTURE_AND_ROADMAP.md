## 0. 설계 철학과 핵심 결정 요약

이 도구는 "그럴듯한 애니메이션"이 아니라 **검증 가능한 수치해석 결과를 3D로 번역하는 엔지니어링 도구**여야 합니다. 따라서 모든 설계 결정은 다음 원칙을 따릅니다.

1. **물리 우선(Physics-first):** 렌더링은 해석 결과의 뷰(View)일 뿐이다. 물리 엔진은 3D와 완전히 분리된 순수 TypeScript 모듈로 작성하고, 교과서/실무 벤치마크(Joukowsky, Crane TP-410 예제, 3-저수지 문제)로 단위 검증한다.
2. **토폴로지와 지오메트리의 분리:** 배관망은 수학적으로는 그래프(노드-엣지), 시각적으로는 3D 씬이다. 두 표현을 단일 소스(Part Catalog + Network Graph)에서 파생시킨다.
3. **메인 스레드는 렌더링 전용:** 수치해석(특히 MOC 과도해석)은 Web Worker에서 고정 시간간격으로 돌고, 메인 스레드는 `SharedArrayBuffer` 스냅숏을 읽어 보간 렌더링만 한다.
4. **실무 규격 그대로:** ASME B36.10 배관 치수, Crane TP-410 K-factor, ISA Cv 특성 곡선, ASME B31.3 응력 판정 — 근사치가 아니라 실제 표 데이터를 카탈로그에 내장한다.

## 1. 물리 모델링 전략

### 1.1 지배 방정식 체계 — 2계층 솔버

시뮬레이터는 **정상상태 솔버(Steady-State)** 와 **과도상태 솔버(Transient)** 를 별도 엔진으로 구현하고, 정상해를 과도해석의 초기조건으로 사용합니다.

#### (A) 정상상태: 비선형 배관망 해석 (Global Gradient Algorithm)

각 노드 i의 연속 방정식과 각 관로 ij의 에너지 방정식:

```
Σ_j Q_ij = D_i                        (노드 유량 보존)
H_i − H_j = r·Q|Q|^(n−1) + m·Q|Q|     (관로 수두손실: 마찰 + 부차손실)
H_pump(Q) = a0 − a1·Q − a2·Q²         (펌프 성능 곡선, 3점 피팅)
```

- **미지수:** 노드 수두 H, 관로 유량 Q → Newton-Raphson으로 동시 풀이. 야코비안은 희소 대칭행렬이며, 네트워크 규모(수백 요소)에서는 직접법(Cholesky)으로 충분.
- **마찰계수 f:** Churchill(1977) 전영역 상관식 — 층류/천이/난류를 단일식으로 처리해 Newton 반복 중 미분 불연속을 회피. 조도 ε는 재질별(탄소강 0.045mm, 스테인리스 0.015mm) 내장.
- **중력/고도차:** 노드마다 표고 z를 가지며 H = z + p/ρg + v²/2g 전수두로 계산. 3D 편집기에서 배치한 Y좌표가 곧 해석 표고가 됨 → **"3D로 조립한 그대로가 해석 모델"**.
- **펌프:** 카탈로그에 H-Q 곡선, 효율 곡선, NPSHr 곡선, 회전관성 I를 갖는 실제형 원심펌프 모델. VFD 속도 제어 시 상사법칙(affinity laws)으로 곡선 스케일링. 체크밸브 내장 옵션.

#### (B) 밸브·피팅 상세 모델 (필수 디테일 요구사항 대응)

**배관 규격 (ASME B36.10, Sch 40 기본 / Sch 80 선택):**

| NPS | OD (mm) | Sch 40 두께 (mm) | Sch 40 ID (mm) |
|---|---|---|---|
| 2" | 60.3 | 3.91 | 52.5 |
| 4" | 114.3 | 6.02 | 102.3 |
| 6" | 168.3 | 7.11 | 154.1 |
| 8" | 219.1 | 8.18 | 202.7 |

**밸브 4종 — 개도(θ)에 따른 Cv 특성 곡선을 카탈로그에 내장하고, 해석 시 K = 0.00214·d⁴/Cv² (d: mm)로 저항 환산:**

| 밸브 | 완전개방 K (Crane) | 고유 특성 | 용도 모델링 |
|---|---|---|---|
| Gate | 8·fT | Quick-opening (개도 30%에 Cv 80% 도달) | 차단용. 부분개도 시 심한 교축 → 공동 위험원 |
| Globe | 340·fT | Linear ~ Equal-percentage | 유량 제어용. 제어 시나리오의 주 액추에이터 |
| Ball | 3·fT | Quick-opening (S자) | 급속 차단 → **수격 트리거의 주범**으로 시나리오화 |
| Butterfly | 30~45·fT (구경별) | Modified equal-percentage | 대구경 제어. σ 공동지수 판정 대상 |

(fT: Crane TP-410의 완전난류 마찰계수, 구경별 표 값 사용. 엘보 90°: 30fT, 45°: 16fT, 티 분기: 60fT, 리듀서: Crane 식 적용)

- 부분 개도 밸브는 시간에 따른 개도 궤적 θ(t)를 받아 매 시간스텝 Cv(θ) → K(t)로 갱신. 이것이 곧 과도해석의 경계조건이 됨.

#### (C) 과도상태: 특성선법(MOC) 수격 해석

1D 압축성 비정상 유동의 연속·운동량 방정식:

```
∂H/∂t + (a²/gA)·∂Q/∂x = 0
∂Q/∂t + gA·∂H/∂x + f·Q|Q|/(2DA) = 0
```

특성선 변환으로 C+/C− 호환 방정식을 얻고, 각 파이프를 Δx = a·Δt로 격자화(Courant 수 = 1 정합):

```
C+: H_P = C_P − B_P·Q_P
C−: H_P = C_M + B_M·Q_P
```

- **압력파 전파속도 a (Korteweg):** `a = sqrt( (K/ρ) / (1 + (K·D)/(E·e)·ψ) )` — 유체 체적탄성계수 K, 관 탄성계수 E, 두께 e, 구속조건 계수 ψ까지 반영. 즉 **같은 물이라도 2" Sch80 강관과 8" Sch40 강관의 파속이 다르게 계산됨.**
- **전 네트워크 공통 Δt:** 최단 파이프 기준으로 Δt 결정, 나머지 파이프는 파속 미세조정(±1% 이내) 또는 보간으로 Courant 정합. 일반적으로 Δt ≈ 0.5~2ms.
- **경계조건 라이브러리:** 저수조(정수두), 밸브(θ(t)→Cv→오리피스식), 펌프(관성 감속 + Suter 4상한 특성곡선으로 펌프 트립/역회전 모사), 체크밸브(역류 감지 슬램 모델), 분기/합류 노드, 서지탱크·에어챔버(제안 기능).
- **비정상 마찰(제안):** 정상 마찰만 쓰면 수격파가 실제보다 덜 감쇠함. Brunone 계수 k₃ 기반 비정상 마찰항을 추가해 파형 감쇠를 실측 수준으로 재현.

#### (D) 공동 현상 (Cavitation) — 2단계 감지 + 1단계 모사

1. **펌프 흡입 공동 (정상/과도 공통):** NPSHa = (p_suction − p_vapor)/ρg + v²/2g 를 매 스텝 계산, 카탈로그의 NPSHr 곡선과 비교. NPSHa < NPSHr + 마진(0.5~1m) 시 경보. 유체 증기압은 온도 의존(Antoine 식, 0~150°C).
2. **밸브 공동 지수:** σ = (p₁ − p_v)/(p₁ − p₂) 를 밸브별 한계 σ_incipient(카탈로그 값)와 비교 — 초생공동/초킹 단계 판정.
3. **관로 증기공동 (DGCM):** MOC 계산점의 압력이 증기압 이하로 떨어지면 Discrete Gas Cavity Model로 국소 공동 체적을 추적. **공동 붕괴 시 재결합 충격압(column separation rejoinder)** — 실제 사고의 주원인 — 까지 재현. 이것이 이 시뮬레이터의 기술적 차별점.

#### (E) 기계적 파괴 응력 판정 (ASME B31.3 기반)

매 시간스텝, 각 파이프 요소에 대해:

- **후프 응력:** σ_h = P·D/(2t) (Barlow). 부식여유 1.5mm 차감한 유효두께 사용.
- **종방향 응력:** σ_L = P·D/(4t) + 자중 굽힘(단순 지지 스팬 근사).
- **판정:** σ_h ≤ S (재질 허용응력: A106-B 137.9MPa 등), 서지 포함 시 B31.3의 단시간 허용 초과(occasional load, 1.33S) 규정 적용 → **"정상운전 OK / 서지 시 위반"을 구분 표시.**
- **활용률(Utilization) = σ/σ_allow** 를 0~1+ 스칼라 필드로 뷰 레이어에 공급 → 색상 렌더링의 데이터 소스.

### 2.2 데이터 모델 — 모듈형 조립 구조

```
PartCatalog (불변 규격 데이터)
 ├─ PipeSpec { nps, schedule, OD, wt, roughness, material(E, S_allow) }
 ├─ ValveSpec { type, nps, CvCurve(θ), K_full, σ_limits }
 ├─ PumpSpec { HQcurve, effCurve, NPSHrCurve, inertia, suterCurve }
 └─ FittingSpec { elbow90/45, tee, reducer → K rules }

NetworkGraph (편집 가능한 인스턴스 모델 — 단일 진실 소스)
 ├─ Node { id, position(x,y,z), type: junction|reservoir|... }
 ├─ Edge { id, from, to, partRef, length, state(θ, rpm...) }
 └─ SubAssembly { id, name, memberIds[], anchorFrame }
      예: "펌프 스키드" = 흡입 스트레이너+블록밸브+펌프+체크밸브+토출밸브

직렬화: 프로젝트 파일(JSON) ← NetworkGraph 그대로 저장/로드
파생 1: SolverInput (worker로 전송되는 평탄화된 수치 배열)
파생 2: SceneModel (InstancedMesh 배치 행렬 + 시각화 속성)
```

- **레고식 조립:** 부품은 표준화된 커넥터(플랜지 페이스, NPS 일치 검사)로 스냅 결합. NPS가 다르면 리듀서 자동 제안.
- **SubAssembly는 재사용 가능한 템플릿:** 한 번 만든 펌프 스키드를 라이브러리에 저장 → 복제 배치 → 전체 네트워크 구성 (End-to-End).

### 2.3 연산 분리 및 상태 관리 (60fps 보장 설계)

```
[Worker]  고정 Δt(≈1ms) MOC 스텝 루프
   │   매 N스텝(≈렌더 프레임당 1회)마다 필드 스냅숏 기록
   ▼
[SharedArrayBuffer 이중 버퍼]
   frameA / frameB + 원자적 세대 카운터(Atomics)
   레이아웃: Float32Array [nodeH..., edgeQ..., utilization..., cavVoid...]
   ▼
[Main] useFrame마다 최신 완성 버퍼를 읽어
   두 스냅숏 사이 선형 보간 → 셰이더 유니폼/텍스처 갱신
```

- **React 리렌더 없음:** 시뮬레이션 필드는 React 상태를 거치지 않고 R3F `useFrame`에서 직접 GPU 자원(DataTexture)에 씀. Zustand는 편집·선택·뷰 모드 등 저빈도 상태만 담당.
- **명령/이벤트 프로토콜:** 메인→워커는 `{type: 'SET_VALVE_TRAJECTORY' | 'PUMP_TRIP' | 'RESET' ...}` 커맨드 큐, 워커→메인은 경보 이벤트(공동 발생, 응력 위반)를 postMessage — 대용량 필드는 SAB, 희소 이벤트는 메시지로 이원화.
- **폴백:** SAB 불가 환경(COOP/COEP 미설정 호스팅)에서는 Transferable ArrayBuffer 핑퐁으로 자동 강등.
- **되감기 링버퍼:** 워커가 최근 30초 분량 스냅숏을 순환 저장 → P9 리플레이 기능의 기반.

### 2.4 3D 뷰 아키텍처 — Global ↔ Detail

**단일 씬 + 카메라/LOD 상태머신** 방식을 채택합니다 (씬 2개 유지보다 전환이 매끄럽고 상태 동기화 문제가 없음).

- **GlobalView:** 전 공정 조망. 파이프는 `InstancedMesh` 실린더(낮은 세그먼트), 밸브·펌프는 심볼릭 저폴리 모델. 압력/응력 컬러맵과 경보 아이콘(빌보드)이 주 정보 채널. 수백~수천 요소에서도 드로우콜 수십 개 유지.
- **DetailView:** 특정 SubAssembly 또는 경보 지점 포커스. 고폴리 파트(플랜지 볼트, 밸브 핸들, 웜기어), 유동 파티클, 공동 버블, 파이프 내부 단면 뷰(클리핑 평면) 활성화.
- **전환:** 더블클릭/경보 클릭 → 카메라 quaternion+position 트위닝(≈0.8s, ease-in-out) + LOD 크로스페이드. 경보 패널의 "위반 항목 클릭 → 해당 지점으로 카메라 플라이투"가 핵심 UX.
- **시각화 레이어 (토글 가능):**
  1. **압력 컬러맵:** 파이프 축방향 1D DataTexture에 H(x) 기록 → 프래그먼트 셰이더가 컬러램프 조회. **수격파가 파이프를 따라 달려가는 것이 색 파면으로 보임.**
  2. **응력 활용률 히트맵:** utilization 0→1을 청→황→적, >1.0은 적색 점멸 + 이미시브.
  3. **유동 파티클:** GPU 파티클을 파이프 중심선 커브 좌표계에서 이류(advect), 속도에 비례한 이동/밀도.
  4. **공동 버블:** DGCM 공동 체적에 비례한 버블 파티클 스폰, 붕괴 시 충격 링 이펙트 + 사운드(선택).
  5. **계측 오버레이:** 가상 계기 부착 지점의 실시간 수치 라벨.

### 2.5 성능 예산

| 항목 | 예산 | 수단 |
|---|---|---|
| 렌더 프레임 | ≤ 16.7ms | InstancedMesh, LOD, frustum culling, 셰이더 기반 색상(머티리얼 교체 없음) |
| 워커 스텝 | 실시간 배속 1× 기준 프레임당 ≤ 12ms | 파이프당 Float64 평탄 배열, 객체 할당 없는 핫루프, 필요시 WASM 이관 여지 확보 |
| 메인↔워커 전송 | 프레임당 0 copy | SharedArrayBuffer |
| 네트워크 규모 목표 | 파이프 500개(≈2만 MOC 격자점) @ 60fps | 위 조합으로 달성 가능 판단 |

## 3. 개발 로드맵 — 7 Phases

각 Phase는 **독립 검증 가능한 산출물**로 끝납니다. (CLAUDE.md의 Goal-Driven Execution 원칙 적용)

### Phase 0 — 프로젝트 스캐폴딩 (0.5일 분량)
- Vite + React + TS + R3F + Zustand + Vitest 셋업, 디렉터리 구조(`src/domain`, `src/physics`, `src/scene`, `src/ui`, `src/worker`), CI 스크립트(typecheck/test/lint).
- **검증:** 빈 3D 씬 + 워커 에코 통신 데모가 뜨고 테스트가 통과.

### Phase 1 — Domain Core + 정상상태 솔버 (물리의 뼈대)
- PartCatalog(배관 4종 규격, 밸브 4종 Cv, 피팅 K, 펌프 곡선, 온도 의존 물성 P6), NetworkGraph, 단위계 모듈.
- GGA Newton-Raphson 솔버: 중력·마찰(Churchill)·펌프 곡선·부분개도 밸브.
- **검증(벤치마크 테스트):** ① Crane TP-410 예제 재현 오차 <1% ② 3-저수지 문제 해석해 일치 ③ 펌프-시스템 곡선 교점 검증.

### Phase 2 — 3D 조립 편집기 + Global View
- 부품 팔레트→스냅 배치, NPS 정합 검사, SubAssembly 생성/저장/복제, 프로젝트 JSON 저장/로드.
- InstancedMesh 렌더링, 정상해 결과의 압력 컬러맵 표시, 펌프 BEP 경고(P4).
- **검증:** 펌프 스키드 모듈을 조립→복제→저수조와 연결해 End-to-End 망 구성, 정상해가 색으로 표시. 파이프 500개 씬 60fps 계측.

### Phase 3 — 과도해석 엔진 (MOC) + Web Worker 통합
- MOC 코어(파속 Korteweg, Courant 격자), 경계조건(저수조/밸브 θ(t)/펌프 트립 Suter/체크밸브), Brunone 비정상 마찰, Joukowsky 예측기(P1).
- SharedArrayBuffer 이중 버퍼 파이프라인, 커맨드 프로토콜.
- **검증(가장 중요):** ① 단일 관로 급폐 시 Joukowsky 압력 상승 이론치 재현 ② 파형 주기 4L/a 일치 ③ Wylie & Streeter 교과서 예제 파형 비교 ④ 워커 실시간 성능 예산 충족.

### Phase 4 — 취약점 분석 + 동적 시각화
- VulnerabilityAnalyzer: B31.3 응력 판정(occasional 1.33S 포함), NPSH 마진, 밸브 σ 지수, DGCM 관로 공동+재결합 충격, 침식 속도(P3).
- 시각화 레이어 5종(§2.4), 경보 패널, 가상 계측 대시보드(P7), 리플레이/슬로모션(P9).
- **검증:** 밸브 급폐 시나리오에서 압력 파면 색상 전파 + 공동 발생/붕괴 + 응력 위반 점멸이 시간 정합되게 표시. DGCM은 Simpson & Bergant 벤치마크와 비교.

### Phase 5 — Detail View, 시나리오 시스템, 방호 설계 루프
- Global↔Detail 카메라 상태머신·플라이투, 고폴리 파트, 단면 뷰.
- 시나리오 편집기(밸브 폐쇄 궤적, 펌프 트립 시퀀스, 수요 변동), 서지 방호 설비 카탈로그(P2: 서지탱크/에어챔버/릴리프).
- **검증:** "취약점 발견 → Detail View 분석 → 에어챔버 설치 → 재해석 → 서지 저감 확인" 워크플로가 완결.

### Phase 6 — 엔지니어링 리포트 & 마감 품질
- 최대/최소 압력 포락선 리포트(P8), 피로 누적 추정(P5, 선택), CSV/이미지 내보내기, 예제 플랜트 3종(급수 부스터, 냉각수 순환, 이송 파이프라인) 번들, 온보딩 투어.
- **검증:** 예제 시나리오 3종이 리포트까지 원클릭으로 재현. 전 Phase 벤치마크 테스트 스위트 그린.

**Phase 간 의존성:** 0→1→2→3→4→5→6 순차이되, 2와 3은 1 완료 후 병행 가능.

## 4. 주요 리스크와 대응

| 리스크 | 대응 |
|---|---|
| MOC 전망(全網) 공통 Δt에서 짧은 파이프가 Δt를 과도하게 줄임 | 최소 길이 규칙(자동 병합) + 파속 ±1% 조정 허용, 필요시 파이프별 보간 MOC |
| SAB 요구 헤더(COOP/COEP) 미지원 호스팅 | Transferable 폴백 경로를 Phase 3에서 함께 구현 |
| 워커 JS 성능 한계(대규모 망) | 핫루프를 할당 없는 평탄 배열로 작성해 두면 Rust/WASM 이관이 국소 교체로 가능 |
| DGCM 수치 진동(공동 체적 채터링) | 표준 완화 기법(가중계수 ψ=1.0, 최소 공동 체적 임계) 적용, 벤치마크로 튜닝 |

# FluidTwin 3D — 산업 판매 수준 슈퍼 업그레이드 설계안 (v2.1)

> 목표: 실제 산업 현장의 배관 엔지니어가 **파이프라인 문제 진단·신규 설계 검토에
> 실무 참고 자료로 쓸 수 있고, 산업체에 유상 판매가 가능한** 수준의 제품.
> 상태: **격차 진단 + 업그레이드 설계 — 승인 후 Phase 7부터 착수**
>
> v2.1 변경: ① 플랜트를 **섹션(구역)으로 분할**하고 섹션별 페이지와 전체
> 페이지가 공존하는 **멀티뷰 플랫폼 아키텍처**를 최우선 Phase로 신설.
> ② **제품 인프라(계정/과금/온보딩)는 나중과제로 후순위화** — 플랫폼과 해석이
> 완성된 뒤 진행.

---

## 0. 요약 (Executive Summary)

현재 FluidTwin 3D는 **"물리 엔진의 뼈대는 올바르게 서 있으나, 제품은 아직
데모"** 단계다. GGA 정상해석·MOC 과도해석·B31.3 판정이라는 골격은 상용 도구
(AFT Fathom/Impulse, Bentley OpenFlows HAMMER, Pipe-Flo)와 같은 계열이고
교과서 벤치마크 검증도 되어 있다. 그러나 판매 가능한 제품과의 거리는 물리
코어보다 **(1) 플랜트 구조화(섹션) 능력, (2) 해석 적용 범위, (3) 엔지니어링
워크플로**에서 훨씬 크다.

| 축 | 현재 | 판매 최소선 | 격차 |
|---|---|---|---|
| 플랜트 구조화 | 평면 네트워크 1개, 뷰 1개 | **섹션 분할 + 섹션별 페이지 + 전체 페이지 동시 제공** | **치명적** |
| 해석 신뢰성·범위 | 물 전용, 4개 구경, 단일 재질, 정상 마찰만 | 다유체·실무 카탈로그·V&V 문서 | **치명적** |
| 엔지니어링 워크플로 | SI 고정, 단일 시나리오, 데이터 교환 없음 | 단위계 전환, 시나리오 비교, EPANET/CSV 연동, 제출용 리포트 | **크다** |
| 3D/UX | 게임식 빌더 (강점이자 약점) | CAD식 정밀 입력 + 지금의 직관성 | 중간 |
| 성능 | 수백 파이프 (dense O(n³)) | 수천 파이프, SAB 스트림 | 중간 |
| 제품 인프라 | 계정·저장·과금·문서 전무 | 클라우드 프로젝트, 라이선스, 온보딩, 법적 고지 | 크다 — **나중과제** |

**전략 제안:** 데스크톱 헤비웨이트(AFT 등, 좌석당 수천$)와 해석 범위로 정면
승부하지 않는다. 차별점은 **섹션 단위로 구조화된 플랜트를 브라우저에서 즉시
전체/부분 뷰로 오가며 보는 3D 플랫폼 + 자동 취약점 스캔**이다. 실제 플랜트는
언제나 구역(Area/Unit)으로 나뉘어 관리되므로, 섹션 구조는 곧 실무자의 머릿속
지도와 도구의 지도가 일치한다는 뜻이다.

## 2. 섹션 기반 멀티뷰 플랫폼 — 핵심 아키텍처 설계 (신규)

실제 플랜트가 Area 100/200/300, 유틸리티동, 펌프실처럼 구역으로 관리되듯,
FluidTwin의 플랜트도 **섹션(Section)의 합**으로 구조화한다. 요구사항:

- 플랜트는 여러 섹션으로 나뉜다. **섹션별 페이지**에서 그 구역만 집중
  편집·해석하고, **전체 페이지**에서 모든 섹션이 합쳐진 플랜트를 조망한다.
- 전체 뷰와 부분 뷰는 항상 **같은 데이터의 두 표현**이다 — 섹션 페이지에서
  고친 것이 전체 페이지에 즉시 반영된다.

### 2.1 도메인 모델 — "단일 그래프 + 섹션 파티션" (분리 그래프 방식 기각)

두 가지 설계 후보 중 **(A) 단일 그래프 + 태깅**을 채택한다.

| 후보 | 방식 | 판정 |
|---|---|---|
| **(A) 단일 그래프 + 섹션 파티션** | `NetworkGraph`는 지금처럼 하나. 노드/엣지가 `sectionId`를 가짐. 섹션 페이지 = 필터된 뷰 | **채택** — 단일 진실 소스 유지, 솔버 무변경, 경계가 자연스럽게 그래프 엣지로 존재 |
| (B) 섹션별 독립 그래프 + 병합 | 섹션마다 그래프를 갖고 전체 해석 시 병합 | 기각 — 경계 노드 이중 정의·병합 충돌·ID 재매핑 등 복잡도가 이득을 압도 |

```
Plant (프로젝트)
 ├─ sections: Section[]        { id, name, color, description }
 ├─ NetworkGraph (단일 진실 소스 — 현행 유지)
 │   ├─ Node { ..., sectionId? }     // 미지정 = "미분류" 가상 섹션
 │   └─ Edge { ... }                  // 엣지 섹션 = from-노드의 섹션 (파생)
 └─ 파생 (저장하지 않고 계산):
     ├─ boundaryEdges: 양끝 노드의 섹션이 다른 엣지 = 섹션 간 연결(타이인)
     └─ sectionSubgraph(id): 해당 섹션 노드 + 내부 엣지 + 경계 엣지 스텁
```

- **직렬화**: 프로젝트 JSON 스키마 버전 상향(`sections[]` + 노드의
  `sectionId`). 구버전 파일은 전 요소 "미분류"로 로드 — 하위호환.
- **경계(타이인)는 저장하지 않는다**: 섹션이 다른 두 노드를 잇는 엣지가 곧
  경계다. 섹션 재배정만으로 경계가 자동 갱신되므로 이중 관리가 없다.
- 기존 `SubAssembly`(펌프 스키드 템플릿)는 섹션 **아래** 계층으로 그대로
  존속 — 섹션은 "구역", SubAssembly는 "장비 묶음"으로 역할이 다르다.

### 2.2 해석 — 전체 해석과 섹션 단독 해석

- **전체 해석 (기본)**: 지금과 동일하게 단일 그래프를 통째로 GGA/MOC 해석.
  섹션은 해석에 영향을 주지 않는 **뷰·집계 단위**다. 결과는 섹션별로
  분해(per-section KPI, 경보 카운트)되어 전체 페이지에 집계된다.
- **섹션 단독 해석 (빠른 반복용)**: 섹션 서브그래프를 추출하고, 잘린 경계
  노드를 **고정수두 경계**로 치환한다. 경계 수두의 기본값은 **최근 전체
  해석의 그 노드 수두**(없으면 사용자 입력 요구). 결과 패널에 "섹션 단독
  해석 — 경계 고정 가정" 배지를 명시해 전체 해석과 혼동을 차단한다.
  - 실무 의미: 한 구역을 고치며 수십 번 재해석할 때 전체 망(수천 파이프)을
    매번 풀 필요가 없고, 경계조건 개념은 엔지니어에게 이미 익숙하다.
  - 과도(MOC) 단독 해석 시 경계는 정수두 저수조로 근사되어 반사 특성이
    실제와 다를 수 있음 — 결과에 경고 문구를 고정 표기하고, 정밀 판정은
    전체 과도해석을 권고한다.

### 2.3 UI — 라우팅과 페이지 구성

SPA 해시 라우팅(`#/plant`, `#/section/:id`)으로 페이지를 분리하되 씬/스토어는
공유한다 (R3F 씬 재생성 없이 필터·카메라 상태만 전환).

**전체 페이지 (`#/plant`)** — 조망과 집계:
- 3D: 전 섹션 렌더. **섹션 색 오버레이 토글**(섹션별 고유색 틴트), 경계
  타이인 마커(◆), 섹션 경계 바운딩 헐(반투명) 표시.
- **섹션 대시보드 그리드**: 섹션 카드마다 KPI(수요, 피크 유속, 최악 B31.3
  활용률, 경보 수, 해석 상태)와 미니 상태색 — 카드 클릭 → 해당 섹션 페이지로
  이동(카메라 플라이투 포함).
- 전체 해석 실행/전체 리포트는 이 페이지 소유.

**섹션 페이지 (`#/section/:id`)** — 집중 작업:
- 3D: 해당 섹션만 불투명 렌더, 인접 섹션은 **고스트(반투명 회색)**로 맥락
  유지, 경계 타이인은 강조. 카메라는 섹션에 자동 핏.
- 빌드 도구는 **활성 섹션에 배치** — 새 요소는 자동으로 이 섹션 소속.
  고스트 노드에 스냅해 연결하면 그것이 곧 타이인(경계 엣지)이 된다.
- 섹션 단독 해석 버튼 + 경계조건 패널(경계 노드 목록·수두 값·출처 표시).
- 섹션 리포트(전체 리포트의 해당 장만) 출력.

**공통 내비게이션**: 상단 브레드크럼(플랜트 ▸ 섹션명) + 섹션 전환
드롭다운 + `[` `]` 단축키로 이전/다음 섹션. 경보 클릭 시 해당 요소가 속한
섹션 페이지로 딥링크.

**섹션 관리자 패널**: 생성/이름·색 변경/삭제(요소는 미분류로 이동),
**선택 요소를 섹션에 배정**(3D 박스 선택 → "Assign to…"), 미분류 요소
카운터 배지(0이 되게 유도).

### 2.4 성능·상태 관리 영향

- Zustand 스토어에 `activeSectionId` + 라우트 상태 추가. 씬은 섹션 필터를
  인스턴스 단위 visibility/색으로 처리(지오메트리 재구축 없음).
- 워커 프로토콜에 `subgraphSolve` 커맨드 추가(노드 화이트리스트 + 경계
  수두 맵) — 기존 솔버 재사용, 입력 추출만 신규.
- 섹션 필터는 인덱스(sectionId → 요소 목록)를 파생 캐시로 유지해 O(1) 전환.

## 3. 업그레이드 설계 — Phase 7–12 (v2.1 재편)

우선순위 원칙: **플랫폼 구조 → 해석 신뢰성 → 워크플로 → 편집기/성능 →
차별화 → (나중과제) 제품 인프라**. 섹션 구조는 도메인 스키마·라우팅·리포트
등 이후 모든 Phase가 딛고 서는 바닥이므로 최우선으로 깐다.

### Phase 7 — 섹션 분할 & 멀티뷰 플랫폼 (신규, 최우선) `[G0a–G0d]` ✅ 구현 완료

> **구현됨.** 단일 그래프 + `sectionId` 태깅(경계 파생), 해시 라우팅 기반
> 플랜트 개요/섹션 페이지, 섹션 색 오버레이·고스트·타이인 마커·카메라 자동
> 프레이밍, 섹션 KPI 대시보드, 섹션 단독 해석(경계 고정수두), 섹션 관리자.
> 예제 4종을 섹션 구조로 재구성, v1 파일 하위호환, 도메인 단위 테스트 12종
> 추가(총 85 그린). UI 전면을 OKLCH 토큰 디자인 시스템으로 재구축.

§2의 설계 전체. 작업 분해와 검증:

1. 도메인: `Section` 타입, `sectionId` 태깅, 파생 경계/서브그래프 유틸,
   직렬화 v-업 + 하위호환 → **검증**: 경계 자동 판별·재배정 시 갱신·구버전
   로드 유닛 테스트.
2. 라우팅 + 전체/섹션 페이지 뼈대, 섹션 관리자, 배정 도구 → **검증**: 예제
   플랜트(냉각수 루프)를 3개 섹션으로 분할하고 페이지 간 왕복·편집 반영이
   즉시 일치.
3. 섹션 렌더 모드(틴트/고스트/타이인 마커) + 섹션 카메라 핏 → **검증**:
   480파이프 그리드에서 섹션 전환이 프레임 드랍 없이 동작.
4. 섹션별 KPI 집계 + 전체 페이지 대시보드 그리드 → **검증**: 전체 해석 후
   섹션 카드 수치의 합이 전역 수치와 일치.
5. 섹션 단독 해석(경계 고정수두) → **검증**: 전체 해석 수두를 경계로 넣은
   섹션 단독 해석이 그 섹션 내부 유량을 전체 해석 대비 ±0.5%로 재현.
6. 예제 플랜트 4종을 섹션 구조로 재구성(예: 냉각수 루프 = 펌프실/공급
   헤더/리턴 헤더).

### Phase 8 — 해석 신뢰성 & V&V `[G1,G2,G4–G9]` (구 Phase 7)

**8a. 다유체 물성 라이브러리** — `domain/fluid.ts`를 유체 레지스트리로 확장.
- 내장: 물(현행), 에틸렌글리콜 수용액(농도별), 해수, 경유(diesel), 윤활유
  ISO VG 32/68, 응축수. 각각 ρ, μ, p_v, K 의 온도 함수. 사용자 정의 유체
  (4개 물성을 상수/테이블 입력).
- 검증: 각 유체 물성을 문헌값 대비 ±2% 이내 스냅숏 테스트.

**8b. 카탈로그 확장** — `domain/catalog/`.
- 배관: NPS ½–24" (½, ¾, 1, 1½, 2, 3, 4, 6, 8, 10, 12, 16, 20, 24),
  Sch 10/40/80/160 (B36.10M/B36.19M 표 데이터).
- 재질: A106-B(현행), A53-B, A312-TP304L/TP316L, HDPE PE4710 —
  각 재질에 **온도별 허용응력 테이블**(B31.3 Table A-1 발췌)과 E(T), 조도.
- 부품: 체크밸브(스윙/노즐 — 역류 슬램 동특성 파라미터), Y-스트레이너,
  오리피스 플레이트, 릴리프 밸브(설정압+블로다운), 진공 브레이커, 서지탱크
  (개방형), 펌프 3계열.
- 검증: 카탈로그 전 항목 치수·K값 스냅숏 테스트, 리듀서 자동 제안이 신규
  구경 조합에서 동작.

**8c. 과도해석 정밀화**
- Brunone 비정상 마찰항(k₃, Vardy-Brown 근사) — 파형 감쇠를 실측 수준으로.
- Suter 4상한 펌프 특성(Wylie & Streeter 무차원 곡선) — 역류·역회전 영역.
- DVCM → **DGCM** 전환(가중계수 ψ, 최소 가스 체적) — 채터링 억제.
- 검증: ① Simpson & Bergant(1994) 컬럼 분리 파형 대비 피크 시각/크기 비교
  ② Wylie & Streeter 펌프 트립 예제 재현 ③ 기존 Joukowsky/4L/a 회귀 유지.

**8d. 수치 코어 강화**
- 희소 솔버: GGA Schur 보완 행렬을 CSR 조립, 희소 Cholesky(또는 Jacobi
  전처리 CG). 목표: **파이프 5,000개 정상해 < 2 s**.
- 수렴 실패 진단기: 잔차 최대 노드, 고립 서브그래프, 제로 유량 경로, 수두
  충돌을 검출해 **3D 하이라이트 + 수정 제안 문구** 출력. 섹션 페이지와 연동
  (문제 요소의 섹션으로 딥링크).
- 검증: 병든 네트워크 6종에서 진단이 정확한 요소를 지목.

**8e. V&V 자동 문서화**
- 벤치마크 스위트가 오차표를 데이터로 출력 → `docs/VALIDATION.md` 자동 생성.
- 제품 내 "Validation" 탭에서 동일 표 노출. Phase 9의 INP import 후 EPANET
  교차 비교를 추가.

### Phase 9 — 엔지니어링 워크플로 (섹션 인지) `[G10–G15]` (구 Phase 8)

- **9a. 단위계**: domain/physics는 SI 불변, 표시 계층 전역 토글(SI ↔ US
  customary) + 단위 접미사 입력 파싱("150 psi"). 검증: 왕복 무손실, 전
  패널·리포트·3D 라벨 반응 스냅숏.
- **9b. 프로젝트 설계 기준**: 설계압력/온도, 적용 코드(B31.3/B31.4/B31.1 —
  판정식 분기), 부식여유, 기본 유체·단위계. `analysis/`가 프로젝트 기준을
  참조하도록 리팩터.
- **9c. Undo/Redo**: 커맨드 스택(스냅숏 + 구조 공유), Ctrl+Z/Y, 50단계 —
  섹션 편집(배정/생성/삭제)도 스택에 포함. 검증: 전 편집 연산 undo→redo
  왕복 동등성 프로퍼티 테스트.
- **9d. 시나리오 매니저**: 복수 시나리오 저장·복제, **비교 뷰**(포락선
  겹치기), **파라미터 스윕**(폐쇄시간 범위 → 피크압 곡선, 워커 배치).
  시나리오는 전체/섹션 단독 해석 양쪽에 적용 가능.
- **9e. EPANET INP import/export**: 서브셋 파서/생성기, 좌표 없으면 자동
  레이아웃. import 시 INP의 태그/패턴을 섹션 후보로 제안. 검증: Net1 왕복
  후 정상해 EPANET 대비 ±1%.
- **9f. 리포트 v2**: 표지(프로젝트 메타데이터·로고), **섹션별 장 구성**
  (전체 요약 → 섹션별 상세), 판정 수식·코드 조항 병기, 포락선·비교 도표,
  print-to-PDF 최적화. 섹션 페이지에서는 해당 장만 단독 출력.

### Phase 10 — CAD급 편집기 & 대형 모델 `[G16,G25,G26,G24]` (구 Phase 9)

- **정밀 입력**: 좌표/길이/각도 키보드 입력, 직교 스냅(45°/90°), 그리드
  설정, 노드 좌표 정밀 편집.
- **모델 브라우저**: 좌측 트리를 **섹션 ▸ 라인/SubAssembly ▸ 요소** 계층으로
  구성(Phase 7 구조를 그대로 노출), 텍스트 검색, 라인 번호 태깅
  (`CW-1201-6"-A1`), 트리 클릭 → 플라이투.
- **뷰 도구**: 단면(클리핑), 두 점 측정, 라인·섹션별 표시/숨김, 경보만 보기.
- **성능**: SAB 이중버퍼 전환(배포 헤더 준비됨), 3D 라벨 거리 컬링/집계,
  목표 **5,000 파이프 @ 60 fps** — 섹션 페이지는 부분 렌더라 자연히 유리.

### Phase 11 — 차별화 기능 `[영업 무기]`

- **자동 취약점 스캔**: 모든 밸브 급폐 + 모든 펌프 트립을 배치 실행 →
  **"가장 위험한 이벤트 Top N"** 순위표(피크압·위반 조항·위치 링크). 전체
  스캔과 **섹션 스캔**(해당 구역 이벤트만) 양쪽 제공.
- **방호 설비 자동 사이징**: 목표 피크압을 만족하는 최소 에어챔버 체적/릴리프
  용량을 이분 탐색으로 제시.
- **리플레이 & 슬로모션**: 워커 링버퍼 30 s, 1/100 배속 + 타임라인 스크러빙.
- **계측 데이터 연동**: 현장 압력/유량 CSV 임포트 → 모델 겹쳐 그리기 + 간이
  보정(조도/국부손실 피팅).
